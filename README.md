# clawd-pet-imessage 🦀

An iMessage bot that sends your iPhone pixel-art GIF reactions whenever Claude Code is working. Inspired by [clawd-on-desk](https://github.com/rullerzhou-afk/clawd-on-desk).

---

## How it works

Claude Code fires lifecycle hooks on every action. Each hook posts an event to a local HTTP server. The server drives a state machine that picks the right GIF and sends it — plus a short text — to your iPhone via AppleScript + Messages.app.

```
Claude Code hooks
    │  (PreToolUse, Stop, SubagentStart, …)
    │  HTTP POST → localhost:23456/state
    ▼
Pet state machine
    │  debounce + cooldown + rate limit
    │
    ├─► GIF select  (assets/gif/<state>.gif)
    │
    └─► AppleScript → Messages.app → iMessage ──► iPhone
```

**Polling loop** (separate thread) reads `~/Library/Messages/chat.db` every 2 s and handles commands you send back from your phone.

## Pet states

| State | Trigger | Auto-returns to |
|---|---|---|
| `idle` | `SessionStart` | `sleeping` after 10 min |
| `thinking` | `UserPromptSubmit` | `idle` after 30 s |
| `working` | `PreToolUse` / `PostToolUse` | `idle` after 60 s |
| `juggling` | `SubagentStart` | `working` after 120 s |
| `sweeping` | `PreCompact` | `idle` after 30 s |
| `happy` | `Stop` (success) | `idle` after 10 s |
| `error` | `PostToolUseFailure` / `StopFailure` | `idle` after 10 s |
| `sleeping` | `SessionEnd` / idle timeout | — |

State transitions respect priority ordering (`error` > `happy` > `sweeping` > `juggling` > `working` > `thinking` > `idle` > `sleeping`). Rapid bursts are collapsed by a 5-second debounce window.

## Commands

Send any of these from your iPhone to control the bot:

| Command | Effect |
|---|---|
| `/pet` | Shows current state + GIF |
| `/status` | State, messages sent today, mute status |
| `/feed` | Forces `happy` state + GIF |
| `/sleep` | Mutes all notifications |
| `/wake` | Unmutes notifications |
| `/help` | Lists all commands |

## Permission Request

When enabled, the bot intercepts dangerous Bash commands before Claude executes them and asks you to approve via iMessage.

**Dangerous patterns that trigger a request:**
`rm`, `rmdir`, `git push`, `git reset`, `git checkout .`, `drop`, `delete`, `truncate`, `kill`, `pkill`, `chmod`, `chown`, `sudo`, `dd if=`

**Reply with:** `y` / `yes` / `ok` to allow — `n` / `no` to deny.

Three modes (configured in `setup.sh`):

| Mode | Behavior |
|---|---|
| OFF | No permission requests, state notifications only |
| Timeout | Waits N seconds; falls back to auto-allow or auto-deny |
| Unlimited | Blocks until you reply (Claude Code pauses) |

## Rate limiting and quiet hours

- **Daily limit:** 100 messages (rolling 24-hour window)
- **Quiet hours:** No messages sent midnight–8 AM (configurable)
- **Debounce:** Rapid state flips within 5 s collapse into one message
- **Per-state cooldown:** Same-state GIF is not re-sent until the cooldown expires (e.g. `working` → 30 s, `sleeping` → 120 s)
- **Manual mute/unmute:** `/sleep` and `/wake` commands

---

## Echo prevention (same-number mode)

When you send messages to your own number, `chat.db` records both sent and received sides. Without protection the bot would re-process its own messages in a loop. Three-layer defense:

1. **Hash cache** — every outgoing message is hashed and stored for 60 s; incoming messages are checked against it.
2. **System-prefix matching** — messages starting with known bot emoji/prefixes are dropped.
3. **SQL filter** — `is_from_me = 0` in the polling query (separate-number mode only).

## Installation

```bash
git clone https://github.com/<user>/clawd-pet-imessage
cd clawd-pet-imessage
./setup.sh
```

`setup.sh` walks you through:

1. Python dependency installation (`Pillow`, `python-dotenv`, `schedule`)
2. Messaging mode — **separate number** (recommended) or **same number** with echo prevention
3. Phone number input (validated as `+<country><digits>`)
4. DRY_RUN mode toggle (logs instead of sending — recommended for first run)
5. Permission Request configuration (off / timeout / unlimited)
6. GIF asset generation (`assets/generate_gifs.py`)
7. Claude Code hook registration (`hooks/install.py`)
8. macOS permission guidance

To start after setup:

```bash
./start.sh
```

`start.sh` re-runs setup automatically if `.env` is missing.

## Configuration

All settings live in `.env` (auto-generated by `setup.sh`). Override any value manually:

| Variable | Default | Description |
|---|---|---|
| `RECIPIENT_PHONE` | — | Phone number in `+821012345678` format |
| `MESSAGING_MODE` | `separate` | `separate` or `same` |
| `SERVER_PORT` | `23456` | Hook HTTP server port |
| `DRY_RUN` | `false` | Log only, no actual sends |
| `DAILY_MESSAGE_LIMIT` | `100` | Max messages per 24 h |
| `QUIET_HOURS_START` | `0` | Quiet period start hour (0–23) |
| `QUIET_HOURS_END` | `8` | Quiet period end hour (0–23) |
| `POLLING_INTERVAL` | `2` | Seconds between chat.db polls |
| `PERMISSION_ENABLED` | `false` | Enable Permission Request feature |
| `PERMISSION_TIMEOUT` | `45` | Seconds to wait for reply; `0` = unlimited |
| `PERMISSION_DEFAULT` | `allow` | Fallback when timeout expires |

## Architecture

```
imessage-pet/
├── main.py               # Entry point; starts HTTP server + polling loop
├── config.py             # Settings loaded from .env
├── setup.sh              # Interactive one-shot installer
├── start.sh              # Run (calls setup.sh if .env missing)
├── requirements.txt      # python-dotenv, Pillow, schedule
│
├── hooks/
│   ├── pet-hook.sh       # Bash hook: curl POST to /state (async, max 1s)
│   └── install.py        # Registers hook in Claude Code settings
│
├── pet/
│   ├── state_machine.py  # Debounce + cooldown + auto-return + idle timer
│   ├── hook_server.py    # HTTPServer on localhost:23456 (/state, /permission)
│   ├── permission_handler.py  # Dangerous-command detection + reply polling
│   └── personality.py    # State messages and command responses
│
├── messaging/
│   ├── sender.py         # AppleScript send_gif() / send_text()
│   ├── chat_db.py        # SQLite reader for ~/Library/Messages/chat.db
│   ├── echo_guard.py     # Three-layer echo prevention
│   └── rate_limiter.py   # Daily limit + quiet hours + mute
│
├── assets/
│   └── gif/              # idle.gif, thinking.gif, working.gif, …
│
└── utils/
    ├── logger.py
    ├── retry.py
    ├── safe_runner.py
    └── state_store.py
```

## DRY_RUN mode

Set `DRY_RUN=true` in `.env` to test without sending any iMessages. All sends are logged to stdout instead. Switch to live mode at any time:

```bash
# .env
DRY_RUN=false
```
